Hacer commits mientras desarrollas

La idea es commitear con frecuencia, especialmente cuando completas una funcionalidad o una parte estable del proyecto.

Buen flujo de trabajo:

Editas tu código en Visual Studio.

Verificas qué archivos cambiaron:

git status

Agregas los cambios al commit:

git add .

Haces el commit con un mensaje claro:

git commit -m "Añadido controlador de reservas"

💡 Tips:

Mensajes claros y cortos ayudan a entender la historia del proyecto.

Commits pequeños y frecuentes son más seguros que commits enormes.
